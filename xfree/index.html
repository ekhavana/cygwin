What is it?

It was my personal idea, to port XFree86, the free and optimized X11R6 implementation to Windows 9x/NT.  Later Chris Faylor, Manager Win32, Red Hat Inc., provided resources by offering access to Red Hat's Sourceware Server and cygwin-xfree mailing list. XFree86 source code and patches for Cygwin are available via FTP from ftp://sourceware.cygnus.com/pub/cygwin/xfree.

One aspect of this project is to port XFree86 to Windows, using Cygwin B20.1. Few years ago, it would not have been feasible to compile XFree86 on Windows. Cygwin B20.1 makes it possible.

John Fortin later joined Cygwin-xfree project. He ported GGI (General Graphical Interface), and DirectX to Cygwin. John also wrote DirectX interface for GGI libraries, as a result XGGI now functions on Windows.

 XFree86 is not just yet another program, it is an entire window management environment, that usually runs in Unix environments. We are talking about  over 100MB of source code, much of which highly hardware dependent, e.g. the X Servers which access the video hardware of the PC directly.

What is the status?

Source code of XFree86 had been patched for Cygwin and new source code to support DirectX had been added to XFree86 source tree. It compiles without problems using Cygwin B20.1  and GCC-2.95.2. Please read "Resource Requirements" for details.

How can I help?

We need contributors for fixing bugs and adding new feature.  One of the desired feature is; integration of X Desktop with Windows Desktop, so X and Win32 can share same desktop.

The patched source is available at ftp://sourceware.cygnus.com/pub/cygwin/xfree

Resource requirements?

  • Hardware:
    • 486, Pentium or PPro system with at least 16 MB main memory
    • To recompile from scratch, 300 MB free space on your hard disk. The system requires a file systems, for example FAT32 and NTFS, which support long filenames.
  • Software and Compilation:
    • Windows 9x/NT 4.0. or Windows 2000.
    • Cygwin B20.1. (A latest Cygwin Development Snapshot is preferred due to improved socket functions, stability and enhanced functionalities).
    • Download XC40-1  to XC40-4.tar.bz2 from ftp://sourceware.cygnus.com/pub/cygwin/xfree/xc-4
    • Please mount Cygwin disk as binary before extracting and compiling XFree86 source code, i.e. umount / then mount -b c: /, otherwise most of the code in xc/programs/Xserver will fail to compile and report "syntax" errors.
    • For network connectivity, you need Windows TCP/IP installed
    • GCC 2.95.2 for Cygwin  (EGCS 1.1.2 may not work because DirectX libraries for Cygwin require GCC 2.95.x).

    • libcrypt to compile xdm security features.  

    • The video support drivers and code from ftp://sourceware.cygnus.com/pub/cygwin/xfree/drivers 

    • John Fortin port of DirectX for Cygwin.  It is available at http://pws.prserv.net/thefortins/cyg-directx.html You need only this if you wish to compile XGGI from source.
    • Install libcrypt in \cygnus\cygwin-b20\H-i586-cygwin32\i586-cygwin32.
    • You may also need Patch.exe. When applying 3 to 6 MB patch file, patch.exe that comes with Cygwin did core dump. I ported GNU patch source code to Microsoft VC.
