What is it?

It was my personal idea, to port XFree86, the free and optimized X11R6 implementation to Windows 9x/NT.  Later Chris Faylor, Manager Win32, Red Hat Inc., provided resources by offering access to Red Hat's Sourceware Server and cygwin-xfree mailing list. XFree86 source code and patches for Cygwin are available via FTP from ftp://mirrors.rcn.net/pub/sourceware/cygwin/xfree.

One aspect of this project is to port XFree86 to Windows, using Cygwin. Few years ago, it would not have been feasible to compile XFree86 on Windows. Cygwin makes it possible. To do this the latest Cygwin is needed. 

John Fortin later joined Cygwin/XFree86 project. He ported GGI (General Graphical Interface), and DirectX to Cygwin. John also wrote DirectX interface for GGI libraries, as a result XGGI now functions on Windows.

XFree86 is not just yet another program, it is an entire window management environment, that usually runs in Unix environments. We are talking about  over 100MB of source code, much of which highly hardware dependent, e.g. the X Servers which access the video hardware of the PC directly.

What is the status?

The source code of XFree86 now has Cygwin support built-in. Further development is still ongoing. XFree86 4.01 supports Cygwin and DirectX. It compiles without problems using the current Cygwin and GCC-2.95.2. Please read "Resource Requirements" for details.

How can I help?

We need contributors for fixing bugs and adding new feature.  One of the desired features is integration of the X Desktop with the Windows Desktop, so X and Win32 can share the same desktop. Currently the X server runs as a full-screen directX application. You can get the source via CVS.

List of archives

  • By default, the URL points to the Redhat Sources FTP server. It contains pre-compiled libraries, servers and clients. The XFree86 4.01 or newer source code available from http://www.xfree86.org/cvs has support for the latest Cygwin. Note: The X-Servers work best on Windows NT or Windows 2000. They should work on Windows Me, and may not work at all on Windows 9x/Me due to 16-bit code in the win 9x GDI - patches for Windows 9x/Me are gladly accepted. 

    X11R6.x precompiled libs and clients:

  • The X11R6.x libs and clients for the latest Cygwin. Download from FTP server.
  • The X11R6.x software is available only for Cygwin 1.1.4 and later. If you have an older installation you will need to update Cygwin using the Install Cygwin Now link on the cygwin home page. Specifically, this means that X11R6 will not work with the old Cygwin B20 release.

Resource requirements?

  • Hardware:
    • 486, Pentium or PPro system with at least 16 MB main memory
    • To recompile from scratch, 300 MB free space on your hard disk. The system requires a file systems, for example FAT32 and NTFS, which support long filenames.
  • Software and Compilation:
    • Required:
    • Windows 9x/NT 4.0. or Windows 2000.
    • The latest Cygwin.  (Avoid the cygwin development snapshots unless you are feeling brave.) While the latest Cygwin Development Snapshot may have new features and support, the development snapshots can and often are broken. The cygwin installed using the Install Cygwin Now link on the cygwin home page. should be reliable.
    • Download the XC40-1  to XC40-4.tar.bz2 from ftp://mirrors.rcn.net/pub/sourceware/cygwin/xfree/xc-4, or use CVS access (http://www.xfree86.org/cvs)
    • Please mount Cygwin disk as binary before extracting and compiling XFree86 source code, i.e. umount / then mount -b c: /, otherwise most of the code in xc/programs/Xserver will fail to compile and report "syntax" errors.
    • For network connectivity, you need Windows TCP/IP installed
    • GCC 2.95.2 for Cygwin  (EGCS 1.1.2 will not work because DirectX libraries for Cygwin require GCC 2.95.x).

    • The video support drivers and code from ftp://mirrors.rcn.net/pub/sourceware/cygwin/xfree/drivers 

    • Optional:
    • John Fortin port of DirectX for Cygwin.  It is available at http://pws.prserv.net/thefortins/cyg-directx.html You need only this if you wish to compile XGGI from source. 
    • You may also need Patch.exe. When applying 3 to 6 MB patch file, patch.exe that comes with Cygwin did core dump. I ported GNU patch source code to Microsoft VC.
