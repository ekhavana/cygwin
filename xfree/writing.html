Cygwin/XFree86
This page is a short document describing my goals and self-proscribed guidelines in managing the Cygwin/Xfree86 site over the last ~6 months. I am writing these up because with a growing number of contributors there is an increasing need for more than one website editor. My time is limited, and that limit should not limit other contributions. - Robert Collins 6 May 2001

I don't recall the exact circumstances under which I became the site maintainer. Suhaib may have asked for help, or perhaps I offered. Certainly the archives will tell the exact story for those of you who are interested. I took up the gauntlet hoping to help reduce the time load on the xfree86 porters, so that they could focus on porting, not on pretty web pages:]. I believe I have been successful in that. 

When I first started editing the pages, they were all slightly different in layout, and content was mixed somewhat between them (as it still is). I started with a small goal:

  • Standard look and feel across all "site" pages.

And from there added

  • W3C HTML conformance, and
  • W3C CSS conformance with
  • no sacrifice in clarity or content.

I then desired to make the page layout feel similar to that of the cygwin project, to whom we owe web space, previous CVS space, and a porting layer than makes Cygwin/XFree86 possible. As a result of that desire the following was done:

  • Copied the Cygwin page layout code.
  • Copied the Cygwin menu layout code.
  • Adapted the Cwygwin menu code to update the news code.

In order to keep the effort required to do all this to a minimum, I adopted a few coding practices:

  1. I used a single site-wide Cascading Style Sheet.
  2. I avoided IE and Netscape specific tags or functionality.
  3. Comment the code to highlight "blocks" of layout/content.
  4. Test the html via the W3C conformance testers. 

While there is still much that can be done to the site, I believe that the above 4 points made it possible with a minimal time investment (and no server-side code).

In addition to the above 4 html-related point I followed a few common-sense writing guidelines. 

  • The use of short, clear sentences.
  • At most 1 statement per sentence.
  • At most 1 concept per paragraph.
  • Attempt to keep the number of styles (font + bold/italic/underline + size) to a maximum of 3 within a given page.
