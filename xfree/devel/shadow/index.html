Cygwin/XFree86 - Shadow Framebuffer Test

Last updated: 29Mar2001 1419 EST

Shadow Framebuffer Test is, you guessed it, a development release.

Shadow Framebuffer Test will work with Windows 95/98/Me, as well as Windows NT 4.0 and Windows 2000.

Colors have now, as of 21Mar2001 2240 EST, been fixed for some colors depths (16, 32 for sure, maybe 24).

Known Issues

  • Your X server, compiled libraries, and fonts must all be the same version, else you will get strange problems (ie. Closing an application window kills the server).
  • Latest test servers are compiled from the XFree86 4.0.3 source, so you must have the Cygwin/XFree86 4.0.3 binaries, fonts, and libraries installed in order to run the most recent test servers.
  • Color depths of 16 and 32 bits per pixel have been tested, 24 bits per pixel may work.
  • Keyboard key mappings are not perfect; however, they are generally correct for US keyboards, as of 28Mar2001. You may find keys that don't work or that are mapped to the wrong symbol.
  • This is not a feature complete release. This is not a stable release.

Installing Shadow Framebuffer Test

Shadow Framebuffer Test is an update to one file, namely XWin.exe, that you install overtop of your current Cygwin/XFree86 installation. Follow the procedures below to backup your original files and install the Shadow Framebuffer Test files. Run the following commands in a Cygwin shell:

cp XWin.exe.bz2 /usr/X11R6/bin
cd /usr/X11R6/bin
mv XWin.exe XWin.orig.exe
bunzip2 XWin.exe.bz2
