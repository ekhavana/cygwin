What Documentation is Available?

Cygwin: A Free Win32 Porting Layer for UNIX Applications

On August 3, 1998, Geoffrey Noer presented this paper at the 1998 Usenix NT Symposium. It provides an overview of the project and also goes into the implementation of Cygwin in some detail. The paper is available in HTML, Word 98, and Postscript formats.

Frequently Asked Questions (FAQ) file

There is a Frequently Asked Questions file for the current beta release in html format. It covers a lot of ground, including installation help, version histories, release notes... Here's a version of the FAQ translated into Japanese.

Cygwin API docs

Some work remains to be done in documenting the Cygwin API. A partial (but useful) list of calls provided by Cygwin are listed in the Frequently Asked Questions file. The list can also be found in the source code in winsup/doc/calls.texinfo. To check if a call is provided but not listed yet, check the source code file winsup/cygwin.din which may be of some use.

It is also possible to distill a fair amount of information from the FAQ and the project mailing lists.

Development Tools

Documentation for the development tools is available from the main Cygnus site in the on-line documentation section.

Writing DLLs

We also have preliminary documents that discuss how to write your own DLLs. You have the choice of creating non-relocatable DLLs or relocatable DLLs. You may also want to read some helpful hints from Gunther Ebert (gunther.ebert@ixos-leipzig.de). And here is some more information, this time from Fergus Henderson . Once you've figured this out, you might want to learn how to use the dl family of calls in your dlls.

Stabs

The tools use a debugging info format called stabs. You can also find this document in GDB (GNU Debugger) distributions. Other preformatted versions (.txt, PostScript) are available from the ftp location.
