Porting UNIX Programs to Win32 with Cygwin

To give you some idea of the power of the Cygwin API, we've put together a list of some of the useful programs that you can compile and run using the Cygwin tools.

The Cygwin tools themselves

Starting with release 17.1, it is possible to self-host, meaning that all the compiler tools can be compiled under both NT and Windows 95 with the release. The rebuilt tools are capable of rebuilding themselves again. This is a very significant test of the tools.

Kerberos 5 server

We've gotten the Kerberos 5 server (250,000 lines of source code!) up and running under Windows NT as of beta 18.

Additional packages that can be built

In addition to the packages mentioned in the above list, the following have also been built with Cygwin by people on the Net:
  1. tcsh 6.06, no source modifications needed (requires b19)
  2. inetutils providing a telnet server for Win 95 and NT running under inetd
  3. X11R6.4 clients for B20
  4. X11R6.3 client libraries (b19) (with this patch)
  5. FreeESS, available here
  6. Lynx 2.5 (b16)
  7. joe 2.8 editor (b16)
  8. GNU Midnight Commander 4.1.5
  9. mutt (mail program)
  10. sharutil-4.2 (b16): with this patch
  11. Autoconf-2.7 (b16)
  12. Texinfo-3.7 (b16) (info itself needs termcap or curses (any offers?))
  13. ncftp by running configure and adding -lreadline to the makefile.
  14. /rdb (http://www.rdb.com). No problems, used b14. Works great.
  15. xdvi - it works with X server installed.
  16. bibtex (Kempson and Aguilar-Sierra's version from the CTAN archives). Specify linux as target, define CC = gcc.
  17. autoconf-2.9
  18. automake-1.0
  19. bc-1.03
  20. pdcurses-2.0
  21. XaoS (X version)
  22. gettext-0.10
  23. texinfo-3.7
  24. shar-utils-4.2: mostly works, but errors on POPEN, patch exists?
  25. ed-0.2: yes, it's true -I want ed! Complains about bcopy
