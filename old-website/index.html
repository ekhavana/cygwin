The Cygwin Project

The Cygwin tools are ports of the popular GNU development tools and utilities for Windows NT, 95, and 98. They function by using the Cygwin library which provides a UNIX-like API on top of the Win32 API.

Use Cygwin to:

  • Port significant UNIX programs to Windows NT/9x without making significant source code changes. Configure and build most GNU software from source using standard Unix build procedures.
  • Work in a fairly complete UNIX-like environment, with access to most common UNIX utilities (from both the provided bash shell and the standard Win32 command shell).
  • Write native Win32 console or GUI applications.

January 4 1999:
EGCS 1.1.1 binaries available, thanks to Mumit Khan! This package replaces the B20 compiler with EGCS 1.1.1 versions of C, C++, ObjC and F77 plus runtime libraries.
December 4 1998:
B20.1 Released! B20.1 is similar to B20 but fixes several very serious bugs in the Cygwin library. A few minor fixes to the tools are also present. We strongly recommend upgrading at least the DLL. See the B20.1 Readme for more information.
November 12 1998:
Two new Cygwin documents are now available:
  • Cygwin Users Guide
  • Cygwin API Reference
November 3 1998:
eCos 1.1 Sources Released. eCos is Cygnus' new open-source, configurable, portable, and royalty-free embedded RTOS. The Cygwin B20 release supports building the eCos tools under Windows NT for all three targets currently available (mn10300-elf, mips-tx39-elf or powerpc-eabi).
October 31 1998:
B20 Released, in time for Halloween! B20 is based on a much-updated version of the Cygwin library. The compiler is EGCS 1.1-based with additional Win32-related patches and optional Mingw support. See the Release Notes for more specifics.
October 27 1998:
We have renamed the project. It is now the Cygwin Project which provides the Cygwin tools which use the Cygwin library (isn't that simpler?). Along with this change, the project has moved to its new home on sourceware.cygnus.com. Please update your links!
