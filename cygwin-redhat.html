So do I have to buy it or what?

The Cygwin DLL is offered under the same license as many other free software packages -- it uses the GNU GENERAL PUBLIC LICENSE (GPL).

This means that if your program uses the Cygwin DLL, you must also provide source code for your program.

There is an alternative to the above free software method of using the Cygwin DLL. Red Hat also provides a "commercial use" license which allows you to distribute only binary copies of your program and the Cygwin DLL. This method requires the purchase of a license from Red Hat.

Obviously, the license that is appropriate for you is entirely based on your requirements. If you cannot distribute source code, then you need the comercial license. If you are able to distribute source code, then the free software license is probably appropriate.

The above information provides links to information about both licensing methods. The purchase of a commercial license will eventually require communication with a Red Hat sales associate.

This person will be able to provide you with information on purchasing a commercial license. They will also be able to help you decide if you need a commercial license.

The sales person will not be able to help you to figure out if you are correctly adhering to the GPL. They are not able to provide advice on the best way to package your installation for adherence to the GPL.

If you have concerns about your distribution and are unsure what needs to be done to adhere to the GPL, you should engage the services of a lawyer. If this is infeasible then you will surely get lots of advice if you send a question to the cygwin mailing list. The advice will, of course, be unofficial and non-binding.


Whichever license you choose, thank you for using Cygwin!
